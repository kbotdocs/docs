<Section>
  특정 레이블의 호출 횟수를 기록하고 로그에 작성합니다.
</Section>
<Section>
  ## 구문

  ~~~js
  console.count();
  console.count(label);
  ~~~

  <Section>
    ### 매개변수

    <ParamList>
      <ParamItem param={"label"} type={"String"} defaultVal={"`\"default\"`"} optional>
        호출 횟수를 기록할 레이블
      </ParamItem>
    </ParamList>
  </Section>
</Section>
<Section>
  ## 설명

  `console.assert()` 메소드는 주어진 조건식이 `false`일 경우에만 로그를 작성합니다. 조건식이 `true`일 경우에는 아무런 로그도 작성하지 않습니다. 이 메소드는 주로 디버깅 목적으로 사용되며, 특정 조건이 충족되지 않을 때 경고 메시지를 출력하는 데 유용합니다.

  예를 들어, 다음과 같이 사용할 수 있습니다:

  ~~~js
  console.assert(1 === 2, "1은 2가 아닙니다.");
  ~~~

  위 코드는 조건식 `1 === 2`가 `false`이므로 아래와 같은 에러 로그를 작성합니다.

  ~~~text
  Assertion failed: 1은 2가 아닙니다.
  ~~~

  만약 조건식이 `true`인 경우, 예를 들어 `console.assert(1 === 1)`을 호출하면 아무런 로그도 작성되지 않습니다. 이는 코드의 특정 부분이 예상대로 작동하는지 확인하는 데 유용합니다.

  한편, `...args`가 전달되지 않는 경우, 아래와 같은 에러 로그가 작성됩니다.

  ~~~text
  Assertion failed
  ~~~
</Section>
