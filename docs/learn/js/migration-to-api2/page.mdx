<Section>
  해당 문서는 레거시 API에서 API2로 마이그레이션하는 방법을 학습하기 위한 자습서입니다. 레거시 API와 API2의 공통점과 차이점을 이해하고, API2로의 마이그레이션을 위한 기본적인 지식을 습득합니다.
  API2의 주요 기능과 사용법을 익히고, 레거시 API에서 API2로의 전환을 원활하게 진행할 수 있도록 돕습니다.

  <Noti type="warning">
    해당 마이그레이션 가이드 과정을 살펴보며 스크립트에 사용 중인 레거시 API의 모든 feature를 API2에서 대체 가능한지 확인한 후 마이그레이션을 결정하는 것을 권장합니다.
  </Noti>
</Section>
<Section>
  ## API2로 마이그레이션해야 하는 이유

  레거시 API에서 API2로 마이그레이션해야 하는 이유는 무엇일까요? 레거시 API는 구현된지 오래된 구식 API입니다. 
  메신저봇R에서는 코드의 유지보수를 어렵게 하고 확장성과 사용성이 충돌하는 등의 이유로 [레거시 API의 단계적 축소를 예고했습니다](https://messengerbotteam.github.io/posts/sunset-legacy-api.html).
  API2는 이러한 문제를 해결하기 위해 설계된 새로운 API로, 더 나은 가독성과 유지보수 경험, 그리고 확장성을 제공합니다.
</Section>
<Section>
  ## 마이그레이션 하기 전에

  본격적으로 마이그레이션을 하기 전, 약간의 설정이 필요합니다.

  1. API2로 마이그레이션할 스크립트의 스크립트 설정에 갑니다.
  2. **레거시 API를 사용합니다.** (더 이상 사용되지 않음)을 선택 해제합니다.

  <Noti type="warning">
    마이그레이션 중 예기치 않은 스크립트의 실행을 방지하기 위해 해당 스크립트의 전원을 끄는 것을 추천합니다.
  </Noti>
</Section>
